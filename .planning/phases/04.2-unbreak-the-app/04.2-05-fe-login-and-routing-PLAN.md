<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input → LoginForm | Username and password collected in form fields; never stored in React state beyond submit handler |
| POST /api/auth/token response → localStorage | access_token written to auth_token key via setToken() |
| ?from= param → navigate() | redirect target must not accept absolute URLs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-42-05-01 | Spoofing | Open-redirect via ?from= param in RequireAuth and apiFetch 401 handler | mitigate | RequireAuth builds from= from location.pathname (same-origin). LoginForm reads from= via useSearchParams then passes to navigate(from, {replace:true}). react-router's navigate() ignores http:// absolute URLs — it only routes within the SPA. Verify: navigate('https://evil.example.com') must result in rendering the root route, not an external redirect. |
| T-42-05-02 | Information Disclosure | password in React state | mitigate | LoginForm.tsx must NOT use useState for the password value. Password lives only in the DOM input's value and the transient FormData snapshot inside onSubmit. No setPassword state. |
| T-42-05-03 | Information Disclosure | session-expired flash ?flash= param renders arbitrary copy | mitigate | SessionExpiredFlash reads searchParams.get('flash') and only renders when the value is exactly 'session_expired'. Any other ?flash= value renders nothing. The flash text is a hardcoded string ("Your session expired — sign in to continue."), not the param value. |
| T-42-05-04 | Spoofing | logout does not revoke JWT on backend | accept | JWT revocation is v2 (AUTH2-01). Mitigated by 60-min exp claim and jwt_secret rotation on breach. Documented in CONTEXT.md Deferred Ideas. |
  | T-42-05-05 | Information Disclosure | username displayed in ChatInterface header | accept | Username is the same value stored in the JWT subject claim and returned by /api/auth/me. It is not a secret. Exposure is intentional (success criterion #2 in ROADMAP Phase 4.2). |
</threat_model>
