<objective>
Wire probe_provider() into the POST /api/chat/session route handler. When the probe returns a ProbeError,
raise HTTPException with the appropriate status code (502 for ollama_unreachable, 400 for all others) and
the structured detail body. Make create_session async if it is not already. Add integration tests that
monkeypatch httpx to exercise all four probe outcomes without a real Ollama.

Purpose: This is the backend half of REQ-llm-provider-ui-fix. The frontend (Plan 06) consumes the
structured error body to render F1–F3 banners. Without this wiring the frontend always receives a
successful session-create response and can never detect provider misconfiguration.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Authenticated client → POST /api/chat/session | Request body contains provider + model (validated against settings allowlist) |
| Backend → Ollama service | Outbound HTTP GET to OLLAMA_BASE_URL/api/tags; URL is server-controlled |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-42-04-01 | Information Disclosure | detail body in HTTPException leaks internal error text | mitigate | detail is probe.model_dump() which is { error, message, hint } — all three values are user-facing copy, not Python tracebacks or internal paths. Verified in RESEARCH.md §"Anti-Patterns". No raw exception strings in the detail. |
| T-42-04-02 | Tampering | provider/model values from request body influence the probe URL | accept | The probe URL is always f"{settings.ollama_base_url.rstrip('/')}/api/tags" — only the server-controlled ollama_base_url is used. provider and model values are only used as string comparisons after the /api/tags response is received. No SSRF vector. |
| T-42-04-03 | Denial of Service | probe runs on every session-create even when provider is None | mitigate | Probe is gated on `if request.provider and request.model` — no probe, no timeout cost when the request omits provider/model. |
| T-42-04-04 | Elevation of Privilege | unauthenticated probe trigger | mitigate | create_session has _current_user: User = Depends(get_current_active_user) — the probe only runs for authenticated users. The 401 check fires before the probe. |
</threat_model>
