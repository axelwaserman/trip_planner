# Plan 04.2-06 Summary — fe-selector-error-banner

## Outcome

`SelectorErrorBanner` ships the F1-F4 selector error surface specified in
`UI-SPEC §"Selector error banner"`: a `role="status" aria-live="polite"`
container with the probe message, optional hint, optional inline `<code>`
chips, and a Retry button. `useChat` and `ProviderSelector` were rewired to
go through `apiFetch`, so all `/api/chat/session`, `/api/chat`, and
`/api/providers` calls carry the bearer token and surface a clean
session-expired redirect on 401. `ChatInterface` merges the F1-F3 probe
errors (from `useChat.providerError`) with the F4 providers-fetch failure
(from `ProviderSelector.onProviderError`) into a single `activeError` and
renders the banner in the header. A post-PR UAT round (`fdd61a9`) deleted
the route's older `available=False` short-circuit so the F3 (missing API
key) signal actually reaches the user — see `04.2-LEARNINGS.md` Decisions
§"Drop the available=False short-circuit". The Wave 0 `test.todo` stubs
for the banner were promoted to real Vitest + RTL assertions in this
completion pass.

## Deviations

**SUMMARY.md was written after PR #7 merge.** The standard order is
plan → execute → SUMMARY → merge. Plan 06 went plan → execute → UAT →
merge → SUMMARY because the checkpoint task in the plan was `autonomous:
false`, the UAT was performed live with the user, the PR was merged on
sign-off, and the formal summary was deferred. Root cause is documented
in `04.2-LEARNINGS.md` Surprise §"Plan 06 was completed but no SUMMARY.md
was written". Future checkpoint plans should write the summary before the
PR squash-merges.

**UAT exposed F3 masking that the Wave 0 test fixture had hidden.** Plan
04's `FakeSettings` test fixture made the `missing_api_key` integration
test pass against the route's older `if not providers[request.provider]
["available"]` short-circuit (which returned a string-detail HTTP 400
*before* the structured `ProbeError` body could fire). In production the
short-circuit fired first, so the banner never saw the structured error.
Fix landed in `fdd61a9`: delete the short-circuit, render every
backend-exposed provider in the dropdown, let the probe own the F1-F4
taxonomy. Cross-references: `04.2-LEARNINGS.md` Decisions §"Drop the
available=False short-circuit" and §"Render every backend-exposed provider
in the dropdown"; Lesson §"Wave-0 test stubs only catch what the planner
thought of".
