## Deviations

**Form element vs Chakra `Box as="form"`.** The plan suggested Chakra's polymorphic
`Box as="form"`. Chakra v3's `Box` types narrow `onSubmit` to `FormEvent<HTMLDivElement>`,
which makes a typed `(e: FormEvent<HTMLFormElement>) => void` handler unassignable.
Switched to a real `<form>` element wrapping a `Stack`. Visual contract unchanged.

**Test stubs left as `test.todo`.** Wave 0 stubs for `RequireAuth.test.tsx` and
`LoginForm.test.tsx` are still `test.todo` (no real assertions). The plan-checker accepted
this earlier; promoting them to real tests adds value but is a separate uplift — left as
follow-up tracked in the existing TODO markers.
