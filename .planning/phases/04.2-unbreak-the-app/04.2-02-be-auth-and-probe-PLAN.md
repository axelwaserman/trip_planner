<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| API layer → Ollama service | probe_provider fires an outbound HTTP GET to OLLAMA_BASE_URL — server-controlled URL, not user input |
| Client → POST /api/auth/token | Public endpoint accepting credentials over HTTPS |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-42-02-01 | Information Disclosure | probe error messages expose OLLAMA_BASE_URL and env-var names | accept | OLLAMA_BASE_URL and ENV_VAR names are configuration metadata, not secrets. F1/F2/F3 messages help users fix their setup. Do not include internal stack traces or file paths. |
| T-42-02-02 | Information Disclosure | provider_probe.py error body surfaced in HTTPException detail | mitigate | Use probe.model_dump() (structured {error, message, hint}) as the detail, not raw Python exception strings. Backend already follows this pattern in routes.py:163. |
| T-42-02-03 | Denial of Service | _probe_ollama hangs without a timeout | mitigate | httpx.AsyncClient(timeout=_OLLAMA_PROBE_TIMEOUT_SECONDS) with _OLLAMA_PROBE_TIMEOUT_SECONDS=1.5 caps the worst case per RESEARCH.md Pitfall 3. |
| T-42-02-04 | Spoofing | POST /api/auth/token brute-force | accept | ADR-009 explicitly drops rate limiting from v1. Documented limitation. Mitigated by Argon2 verification latency (~50-100ms). |
| T-42-02-05 | Spoofing | JWT_SECRET defaults to "changeme" | accept | settings.model_post_init emits a warning at startup when the default is in use. Production deploys must override JWT_SECRET. No new code change needed — existing warning is the control. |
</threat_model>
