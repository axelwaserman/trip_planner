## Decisions Made

- **Frontend stubs use test.todo (no callback) over `describe.skip` + dynamic imports.** Reason: react-router-dom is not yet installed (Plan 03 adds it) and the auth/providerErrors modules + RequireAuth/LoginForm/SelectorErrorBanner/UserMenu components do not exist yet (Plans 03/05/06 create them). test.todo accepts a name string with no callback, so Vitest reports it as TODO without ever evaluating any imports for the assertion body. This avoids both module-not-found errors (no missing-module imports) and the dynamic-import-inside-callback complication that would have been required if we used describe.skip.
- **Backend stubs use `xfail(strict=False)` over `pytest.skip()`.** Reason: skip is invisible in the test count — an executor could land Plan 02 and never notice the gate didn't go green. xfail with strict=False reports XPASS loudly when the implementation lands, prompting the executor to remove the marker.
- **Conditional `try/except ImportError` shield in test_provider_probe.py.** Reason: app.services.provider_probe genuinely does not exist on disk yet, and pytest collection imports the file unconditionally. The shield lets collection succeed; the per-test `assert probe_provider is not None` gate then fails fast (under xfail) so the test never silently passes against a missing module.
- **Type-ignore comment placement.** mypy strict in this repo enables `warn_unused_ignores`; an `# type: ignore[import-not-found]` on the missing import is **needed** to silence the import-not-found error today, but mypy's quirk with module-level conditional imports flags it as "unused" in some forms. The narrow `# type: ignore[import-not-found]` placement that survives strict mode was confirmed by running mypy after each edit. The `probe_provider = None` fallback assignment did NOT need an ignore (mypy infers `None` cleanly when the import branch is unreachable for it).
